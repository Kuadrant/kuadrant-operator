# OLMv1: Child Operator Repo Cleanup

When child operators are distributed solely via their Helm chart (pulled into the kuadrant-operator umbrella), the OLM packaging artefacts in each child operator repo become unnecessary. This document uses dns-operator as the reference example — the same applies to authorino-operator and limitador-operator.

## What Can Be Removed

### Directories (delete entirely)

| Directory | Purpose | Why remove |
|-----------|---------|------------|
| `bundle/` | OLM bundle (CSV, CRDs, metadata, scorecard tests) | Generated by `operator-sdk generate bundle`, OLM-only |
| `catalog/` | OLM catalog channel entry templates | Used only by `generate-catalog.sh` |
| `config/manifests/` | Kustomize overlay for OLM bundle generation | References CSV base, samples, scorecard — all OLM |
| `config/deploy/olm/` | OLM install manifests (CatalogSource, Subscription, OperatorGroup) | Not needed when installed via umbrella operator |
| `config/scorecard/` | OLM scorecard test configuration | OLM-specific testing framework |

### Files (delete)

| File | Purpose |
|------|---------|
| `bundle.Dockerfile` | Builds OLM bundle image |
| `utils/generate-catalog.sh` | Generates OLM catalog from bundle images |
| `make/catalog.mk` | All catalog make targets |

### Makefile Targets (remove)

| Target | Purpose |
|--------|---------|
| `bundle` | Generate OLM bundle manifests |
| `bundle-build`, `bundle-push` | Build/push bundle image |
| `bundle-post-generate` | Post-process bundle (OpenShift annotations, dedup) |
| `bundle-ignore-createdAt` | Work around operator-sdk createdAt drift |
| `bundle-operator-image-url` | Read image from generated CSV |
| `manifests-gen-base-csv` | Update CSV base template |
| `catalog`, `catalog-build`, `catalog-push` | Generate/build/push OLM catalog |
| `catalog-dockerfile` | Generate catalog Dockerfile |
| `deploy-catalog`, `undeploy-catalog` | Deploy/remove OLM CatalogSource |
| `install-olm`, `uninstall-olm` | Install/remove OLM framework |
| `verify-bundle` (in verify.mk) | CI verification of bundle generation |

### Makefile Variables (remove)

| Variable | Purpose |
|----------|---------|
| `CHANNELS`, `DEFAULT_CHANNEL` | OLM channel metadata |
| `BUNDLE_CHANNELS`, `BUNDLE_DEFAULT_CHANNEL`, `BUNDLE_METADATA_OPTS` | Bundle generation flags |
| `BUNDLE_IMG`, `BUNDLE_GEN_FLAGS`, `USE_IMAGE_DIGESTS` | Bundle image config |
| `OPERATOR_SDK`, `OPERATOR_SDK_VERSION` | operator-sdk tool (OLM-only use) |
| `OPM`, `OPM_VERSION` | opm tool (catalog generation only) |
| `CATALOG_IMG`, `CATALOG_FILE`, `CATALOG_DOCKERFILE` | Catalog image config |

### Makefile Target Modifications

| Target | Change |
|--------|--------|
| `prepare-release` | Remove `$(MAKE) bundle` line |
| `helm-build` | Remove `operator-sdk` prerequisite |
| `verify-all` | Remove `verify-bundle` from dependencies |

### GitHub Workflow Changes

| Workflow | Change |
|----------|--------|
| `build-images.yaml` | Remove `build-bundle` and `build-catalog` jobs |
| `build-images-for-tag-release.yaml` | Remove `build-bundle`, `build-catalog`, `verify-builds` jobs |
| `ci.yaml` | Remove `verify-bundle` job and from `required-checks` |

### Documentation

| File | Change |
|------|--------|
| `docs/RELEASE.md` | Remove OLM verification section, update generated files list |

## Kustomize Layer Relationship

The Helm chart and OLM bundle share the same source manifests via different kustomize overlays. Removing the OLM overlay does not affect the Helm chart generation.

```
config/crd/     ─┐
config/rbac/    ─┼─> config/default/ ─┬─> config/helm/      → Helm chart (KEEP)
config/manager/ ─┘                    ├─> config/deploy/     → make deploy (KEEP)
                                      └─> config/manifests/  → OLM bundle (REMOVE)
```

The shared layers (`config/crd/`, `config/rbac/`, `config/manager/`, `config/default/`) are generated by `controller-gen` from kubebuilder markers and are needed by both Helm and local dev paths. They stay.

## What Remains

### Core (unchanged)

```
cmd/                    # Entry point
api/v1alpha1/           # CRD types
internal/               # Controllers, providers
pkg/                    # Shared packages
test/                   # Tests
Dockerfile              # Operator image
Makefile                # Build targets (cleaned)
```

### Kustomize (unchanged)

```
config/crd/             # CRD bases (generated by controller-gen)
config/rbac/            # RBAC (generated from kubebuilder markers)
config/manager/         # Deployment + metrics service
config/default/         # Composition overlay (crd + rbac + manager)
config/deploy/local/    # Local dev overlay (patches for Kind)
config/samples/         # Example CRs
```

### Helm Chart (unchanged)

```
charts/dns-operator/
├── Chart.yaml
├── values.yaml
├── templates/
│   └── manifests.yaml    # Generated from config/helm/ overlay
└── .helmignore
```

### Build Pipeline (simplified)

```
config/helm/            # Kustomize overlay for Helm chart generation
make/helm.mk            # Helm build/install/package targets
make/verify.mk          # verify-manifests, verify-helm-build (no verify-bundle)
```

## Proposed Repo Layout

```
dns-operator/
├── api/v1alpha1/               # CRD types
├── cmd/main.go                 # Entry point
├── internal/                   # Business logic
│   ├── controller/
│   └── provider/
├── pkg/                        # Shared packages
├── test/e2e/                   # Tests
├── config/
│   ├── crd/                    # Generated CRDs
│   ├── rbac/                   # Generated RBAC
│   ├── manager/                # Deployment manifest
│   ├── default/                # Kustomize composition
│   ├── deploy/local/           # Local dev overlay
│   ├── helm/                   # Helm kustomize overlay
│   └── samples/                # Example CRs
├── charts/dns-operator/        # Helm chart (the distribution artefact)
├── make/
│   ├── helm.mk                 # Helm targets
│   ├── kind.mk                 # Local cluster targets
│   └── verify.mk               # CI verification
├── Dockerfile
├── Makefile
└── docs/
```

## Impact Summary

| Metric | Before | After |
|--------|--------|-------|
| Config directories | 10 | 5 |
| Make targets (OLM-related) | ~15 | 0 |
| Make variables (OLM-related) | ~12 | 0 |
| CI jobs (OLM-related) | 4 | 0 |
| External tool dependencies | operator-sdk, opm, kustomize, yq, helm | kustomize, yq, helm |
| Distribution artefacts | OLM bundle image, OLM catalog image, Helm chart | Helm chart |
